Undefined, null i NaN a JavaScript

1. Què és undefined?

Quan una variable és declarada però no té valor assignat, JavaScript li dona automàticament el valor undefined. També pot aparèixer si una funció no retorna res explícitament. 

        
            let x;
            console.log(x); // undefined

            function saluda() {
                console.log("Hola!");
            }
            let resultat = saluda(); // undefined perquè no hi ha return

2. Què és null?

El valor null és assignat de manera intencionada per indicar que una variable no té cap valor. És un valor "buit" explícit. 

        
            let usuari = null;
            console.log(usuari); // null

3. Què és NaN?

Significa “Not a Number” (no és un número). Apareix quan intentes fer una operació matemàtica amb un valor que no es pot convertir a número. 

        
            let resultat = "hola" * 3;
            console.log(resultat); // NaN

            let nom = "Anna";
            console.log(Number(nom)); // NaN

4. Exercicis ràpids

  • Declara una variable sense assignar i mostra el seu valor.
  • Assigna null a una variable i mostra-la.
  • Multiplica una cadena no numèrica per un número i observa el resultat.
        
            let a;
            console.log(a); // undefined

            let b = null;
            console.log(b); // null

            let c = "gat" * 2;
            console.log(c); // NaN

Conclusió

undefined indica que una variable no té valor assignat automàticament. null és un valor assignat manualment per indicar absència de valor. NaN apareix quan intentem fer càlculs amb valors no numèrics. És important diferenciar-los per evitar errors comuns en el teu codi.
