Tipus de dades a JavaScript

En JavaScript, els tipus de dades es divideixen en dades primitives i en objectes. Les dades primitives són els valors més bàsics que pot gestionar el llenguatge. No són objectes i no tenen mètodes. Cada dada té un tipus específic que indica com s’ha de tractar. Entendre els tipus de dades és essencial per programar correctament.

Dades de tipus primitiu

Els tipus primitius de dades són immutables (no es poden modificar directament) i es guarden per valor.

  1. Cadena de text | String

    2. Són cadenes de text. Es poden escriure entre cometes simples, dobles o inclinades (`backticks`). Exemple: "Hola" o 'JS'. 

                
                let a = "Hola";
                console.log(a); // "Hola"

  2. Número | Number

    3. Són números, ja siguin enters o decimals. Exemple: 42 o 3.14. No hi ha diferència entre nombres enters i decimals a JS. NaN (Not a Number) és de tipus number. 

                
                let a = 10.3;
                let b = 3;
                let resultat = a + b;
                console.log(resultat); // 13.3

  3. Booleà | Boolean

    4. Només pot tenir dos valors: true (cert) o false (fals). S'utilitza per fer comprovacions i condicions. 

                
                let a = 4;
                let b = 6;
                let resultat = a > b;
                console.log(resultat); // False

  4. Undefined

    5. És el valor que té una variable quan l’hem declarat però encara no li hem assignat cap valor. Exemple: let x; → x és undefined. 

                
                let a;
                console.log(typeof(a)); // 1

  5. Symbol

    6. És un tipus especial de dada que crea valors únics. S'utilitza menys sovint i és útil per identificar propietats úniques d'objectes.

  6. Null

    7. Representa una absència de valor intencionada. L’hi donem nosaltres per dir que una variable està buida de manera voluntària: let resultat = null;

  7. BigInt

    8. Permet treballar amb números molt grans, més grans del que permet el tipus Number. S’escriu afegint una “n” al final: 123456789012345678901234567890n

Funcions per convertir dades

A vegades necessitem canviar el tipus de dada, i per això tenim funcions que ens ajuden:

  • String( ): converteix qualsevol valor en una cadena de text. Exemple: String(123) retorna "123"
  • Number( ): converteix una cadena o booleà en número. Exemple: Number("5.5") retorna 5.5
  • parseInt( ): converteix una cadena a número enter. Exemple: parseInt("42.7") retorna 42
  • parseFloat( ): igual que parseInt però manté els decimals. Exemple: parseFloat("42.7") retorna 42.7
  • Boolean( ): converteix valors a true o false. Exemple: Boolean(0) retorna false, Boolean("Hola") retorna true

Objectes creats a partir de tipus primitius

Encara que les dades primitives com String, Number o Boolean no siguin objectes, JavaScript pot crear objectes temporals a partir d’elles per poder utilitzar mètodes útils. Per exemple, podem aplicar toString() per convertir qualsevol valor en una cadena de text. 

        
            // Exemple amb un número
            let edat = 25;
            let edatComText = edat.toString(); // "25"

            // Exemple amb un booleà
            let actiu = true;
            console.log(actiu.toString()); // "true"

També es poden crear objectes explícits amb constructors, però no és gaire recomanable fer-ho amb dades primitives: 

        
            let text = new String("Hola"); // Objecte String
            let numero = new Number(123);  // Objecte Number
            let cert = new Boolean(true);  // Objecte Boolean

Aquests objectes es comporten com les seves versions primitives, però ocupen més memòria i poden donar comportaments inesperats. És millor treballar amb valors literals com "Hola", 123 o true.

Diferència entre .toString() i String()

String(valor) és una funció global que converteix qualsevol valor a cadena de text. És ideal quan no estàs segur del tipus que tens (pot ser null, undefined, etc.) perquè no dóna error.

.toString(valor) és un mètode d'objecte, o sigui, només el pots cridar si el valor ja és un número, booleà, string, array, etc. ⚠️ Dona error si el valor és null o undefined perquè aquests no tenen mètodes.

Objectes

Els objectes són una estructura clau a JavaScript per emmagatzemar dades relacionades en forma de clau:valor. A diferència dels tipus primitius, els objectes poden contenir molts valors dins d’una sola variable, fins i tot funcions. 

        
            let persona = {
                nom: "Anna",
                edat: 25,
                saludar: function() {
                  console.log("Hola!");
                }
              };

Cada propietat (com nom o edat) es pot accedir amb . o ["clau"], i pots modificar-les o afegir-ne de noves. Els objectes són mutables i es passen per referència, per això quan els canvies dins d’una funció, els canvis es mantenen fora també. Són la base per a arrays, funcions, dates, JSON i gairebé tot a JavaScript.

Conclusió

Els tipus de dades a JavaScript es divideixen en primitius i objectes, i cadascun es comporta de manera diferent. Els primitius són simples, immutables i es comparen pel valor; en canvi, els objectes són complexos, mutables i es comparen per referència. Conèixer aquestes diferències ens ajuda a evitar errors habituals en comparacions, conversions i manipulació de dades. A més, funcions com String(), Number() o Boolean() ens permeten transformar valors de manera segura i controlar millor el comportament del codi.
